Modelado de Datos

Documentación del proceso de extracción, limpieza y transformación del dataset de campañas de marketing bancario. Responsable: Marco Antonio Cruz.

01

Acerca del Dataset

Los datos están relacionados a campañas de marketing bancario orientadas a la oferta de depósitos a término fijo. El dataset permite analizar el perfil demográfico de los clientes, sus variables financieras, la estrategia de contacto utilizada y la variable objetivo de aceptación o rechazo de la oferta.

Público objetivo: gerentes de marketing bancario y público general interesado en decisiones basadas en datos. El análisis utiliza un lenguaje claro, estratégico y no excesivamente técnico.

Intención narrativa: el proyecto busca demostrar que la segmentación inteligente mejora la eficiencia de campañas comerciales. Entender el perfil del cliente es más esencial que aumentar el número de llamadas.

Variables Originales

La base de datos cuenta con 17 variables: 7 de tipo numérico entero y 10 de tipo categórico (objeto). A continuación se describen todas las variables.

04

Revisión de Variables Categóricas

Se analizaron las 10 variables categóricas mediante gráficos de barras para verificar que contaran con categorías representativas y que no presentaran un único valor dominante. Las frecuencias se calcularon con value_counts().

Python for col in catcols: plt.figure(figsize=(10, 5)) ax = df[col].value_counts().plot(kind='bar', color='skyblue', edgecolor='black') ax.set_title(f'Conteo por categoría: {col}') ax.set_xlabel(col) ax.set_ylabel('Frecuencia') plt.xticks(rotation=45, ha='right') plt.tight_layout() plt.show()

Criterio de evaluación

Una variable fue candidata a eliminación si una categoría era dominante con más del 95% de participación, o si no aportaba variabilidad relevante al análisis. Cada variable se evaluó individualmente considerando su contexto.

Hallazgos destacados

  • default: el 98% de los registros corresponden a una misma categoría ("no"). Sin embargo, al hacer referencia a historial de mora, se conservó por su relevancia para caracterizar el perfil del cliente.
  • poutcome: cuenta con 4 categorías, donde la mayoría corresponde a "unknown" u "other", lo que limita su valor informativo. Se conservó pero se tuvo en cuenta esta limitación en el análisis.

05

Revisión de Variables Continuas

Se realizó un análisis de valores atípicos mediante boxplots e histogramas. El criterio combinado buscó identificar límites estadísticos sin eliminar información relevante para el análisis.

Las variables age, day y balance se excluyeron de la limpieza: la edad tiene un rango válido (18–95), los días se distribuyen correctamente dentro del mes, y balances altos representan clientes de alto valor para el banco.

Variables Derivadas para Visualización

Se crearon variables derivadas para facilitar el análisis visual, mejorar la interpretación para usuarios no técnicos y optimizar la construcción del dashboard en Looker Studio.

Acepta_bin
Variable binaria creada a partir de y. Facilita el cálculo de tasas de aceptación. El promedio de esta variable estima directamente la probabilidad de aceptación dentro de cada segmento.
1 = Cliente aceptó el depósito 0 = Cliente no aceptó
Looker Studio CASE WHEN y = "yes" THEN 1 ELSE 0 END
Grupo_edad
Variable categórica construida a partir de age para analizar el comportamiento por ciclo de vida. Permite identificar patrones diferenciales de aceptación según etapa de vida.
18–29 30–39 40–49 50–59 60+
Looker Studio CASE WHEN age < 30 THEN "18-29" WHEN age < 40 THEN "30-39" WHEN age < 50 THEN "40-49" WHEN age < 60 THEN "50-59" ELSE "60+" END
Grupo_contactos
Variable categórica derivada de campaign. Agrupa el número de contactos durante la campaña actual para evaluar si más intentos se asocian con cambios en la probabilidad de aceptación.
1 contacto 2–3 contactos 4–6 contactos 7+ contactos
Looker Studio CASE WHEN campaign = 1 THEN "1 contacto" WHEN campaign BETWEEN 2 AND 3 THEN "2-3 contactos" WHEN campaign BETWEEN 4 AND 6 THEN "4-6 contactos" ELSE "7+ contactos" END
Canal_contacto
Variable categórica derivada de contact. Traduce los canales de contacto al español para facilitar la lectura en el dashboard.
Celular Teléfono Desconocido
Looker Studio CASE WHEN contact = "cellular" THEN "Celular" WHEN contact = "telephone" THEN "Teléfono" ELSE "Desconocido" END
